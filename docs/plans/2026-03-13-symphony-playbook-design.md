# Symphony Playbook Bootstrap Design

**Goal:** Create a lightweight GitHub repository under `~/Documents/GitHub/symphony-playbook` for personal Symphony workflow notes, starting with a dated bilingual log rather than a fully distilled knowledge base.

**Why this shape:** The current need is not a polished framework. It is a place to capture real questions, code-reading results, and working conclusions without over-abstracting too early.

## Chosen Direction

- Repository type: lightweight research repo
- Language: bilingual (`中文 + English summary`)
- Content style: time log plus lightweight index
- First entry: the Linear + Symphony review/rework decision question

## Minimal Information Architecture

- `README.md`: positioning, structure, and top-level index
- `notes/`: dated logs
- `cases/`: later distilled reusable patterns
- `templates/`: a note template for future entries
- `docs/plans/`: repo bootstrap docs

## Quality Bar

- The repo should be easy to append to.
- The first note should preserve the original question.
- The note should include the key Symphony files that were inspected.
- The note should end with a concrete recommendation, not just exploration.
